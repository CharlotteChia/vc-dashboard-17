# 📊 Enterprise Revenue Intelligence Dashboard

A high-performance, modern SaaS sales analytics dashboard built with **React**, **Vite**, and **Tailwind CSS**. Designed for real-time monitoring of revenue, order volume, and profit metrics.

![Dashboard Preview](https://raw.githubusercontent.com/lucide-react/lucide/main/icons/bar-chart-3.svg)

## ✨ Features

- **Live KPI Tracking**: Real-time cards for Gross Revenue, Total Orders, Net Profit, and Average Order Value (AOV).
- **Intelligent Trends**: Visual indicators for performance growth or decline compared to previous periods.
- **Detailed Audit Log**: A comprehensive, searchable table of all sales transactions filtered by channel and product.
- **Premium UI/UX**: 
  - Glassmorphic design with subtle transparency.
  - Vibrant gradient accents and modern typography.
  - Fully responsive layout for desktop and tablet views.
- **Seamless Data Integration**: Automatic parsing of local CSV data using `PapaParse`.

## 🚀 Tech Stack

- **Framework**: [React](https://reactjs.org/)
- **Build Tool**: [Vite](https://vitejs.dev/)
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/) with PostCSS
- **Icons**: [Lucide React](https://lucide.dev/)
- **Data Parsing**: [PapaParse](https://www.papaparse.com/)

## 🛠️ Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- npm or yarn

### Installation

1. Clone the repository or download the source code.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Setup your environment variables:
   - Create or edit the `.env` file in the root directory.
   - Add your Gemini API key:
     ```env
     VITE_GEMINI_API_KEY=your_actual_key_here
     ```
4. Build environment:
   ```bash
   npm install -D @tailwindcss/postcss
   ```

### Running Locally

To start the development server:
```bash
npm run dev
```
The dashboard will be available at `http://localhost:5173/`.

## 📁 Data Source (CSV & Database)

The dashboard is configured to read data from a local CSV (`/data/sales_data.csv`) by default, but it is fully ready to connect to **Supabase** whenever you want to upgrade to a real database.

### 🚀 How to Connect to Supabase (Easy Mode)

1.  **Create Supabase Project**: Go to [Supabase](https://supabase.com/) and create a new project.
2.  **Create the Table**: In your Supabase dashboard, go to the **SQL Editor** and run this snippet to create the `sales_data` table:
    ```sql
    CREATE TABLE public.sales_data (
        id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
        date DATE NOT NULL,
        product TEXT NOT NULL,
        channel TEXT NOT NULL,
        orders INTEGER DEFAULT 0,
        revenue NUMERIC DEFAULT 0,
        cost NUMERIC DEFAULT 0,
        visitors INTEGER DEFAULT 0,
        customers INTEGER DEFAULT 0,
        created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
    );
    ```
3.  **Import Data**: Use the **Supabase Table Editor** to import your CSV file directly into this new table.
4.  **Get Your Keys**: Go to **Project Settings -> API** in Supabase to find your **Project URL** and **Anon Key**.
5.  **Edit your .env file**: Update your local `.env` file with your credentials and tell the engine to use Supabase:
    ```env
    DATA_SOURCE=SUPABASE
    SUPABASE_URL=your_project_url_here
    SUPABASE_ANON_KEY=your_anon_key_here
    ```
6.  **Restart**: Stop and restart your development server. The dashboard will now serve data directly from your Supabase database!
